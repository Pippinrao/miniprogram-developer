# 技能质量自检清单

> 在新技能首次发布前和每次大改后逐项检查。

## 结构完整性

- [ ] SKILL.md 存在，角色声明为「调度协调者」
- [ ] 工作流选择表不为空，每个关键词映射到具体工作流
- [ ] 子Agent分发表不为空，每个Agent有对应文件
- [ ] 协议文件 `protocols/context-exchange.md` 存在
- [ ] `agents/` 目录下每个文件包含6段（角色/输入/执行/输出/约束/示例）
- [ ] `workflows/` 目录下每个文件包含7段（触发/流程/阶段/决策/错误/验收/占位符）
- [ ] `references/` 目录下有 `knowledge-retrieval.md`
- [ ] `README.md` 存在，包含快速开始指南

## 协议合规

- [ ] 主→子消息 ≤500 tokens（通过估算验证）
- [ ] 子→主消息 ≤300 tokens（通过估算验证）
- [ ] 所有Agent的输出JSON包含 `status, summary, filesChanged, keyFindings`
- [ ] `testResults` 字段：非测试Agent可为null
- [ ] `artifacts` 字段：无产物时为空数组
- [ ] 所有Agent的输出与协议字段名一致（无 `filesCreated` vs `filesChanged` 混淆）

## 交叉引用

- [ ] SKILL.md 中引用的所有工作流文件存在
- [ ] SKILL.md 中引用的所有Agent文件存在
- [ ] SKILL.md 中引用的所有参考文件存在
- [ ] 工作流中引用的Agent名称与SKILL.md的Agent表一致
- [ ] 协议中引用的路径存在

## API准确性

- [ ] 所有代码示例中的API/方法名称经过真实文档验证
- [ ] 无不存在的API（fabricated APIs = 0）
- [ ] 无不正确的API签名（如错误的参数名/类型）
- [ ] 不存在的API被正确标注为不可用（而非直接使用）
- [ ] 所有组件库名称和路径已验证

## 安全性

- [ ] 无硬编码的密钥/Token/密码
- [ ] 错误信息不泄露内部实现细节（如文件路径、堆栈跟踪）
- [ ] 依赖版本号有合理范围（不使用 `latest`）
- [ ] 敏感操作有权限校验

## 内容真实性

- [ ] `official-docs/` 中的文档来自官方来源
- [ ] 文档页数声明与实际文件数一致
- [ ] 无AI编造的文档内容
- [ ] 所有文档标注了来源URL和爬取时间

## 实用性

- [ ] 工作流描述可直接执行（非抽象概念）
- [ ] 代码示例可直接或少量修改后使用
- [ ] 错误处理表覆盖主要的失败场景
- [ ] Token预算在任何场景下都不会超限

## 盲审评分

- [ ] 综合评分 ≥ 7.0 (推荐 ≥ 8.0)
- [ ] Critical 问题 = 0
- [ ] Major 问题 ≤ 2
- [ ] 交叉引用断裂 = 0
- [ ] 捏造 API = 0
