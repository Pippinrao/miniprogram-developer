# 多 Agent 协作架构模式

## 核心设计

```
┌─────────────────────────────────────────────────────────┐
│                    主调度 Agent                         │
│              (Coordinator / Orchestrator)               │
│                                                         │
│  职责: 意图识别 → 工作流选择 → 阶段管理 → 派发子Agent  │
│  不做: 读代码 / 写代码 / 分析 / 测试 / 修改            │
│                                                         │
└──────────┬──────────┬──────────┬──────────┬────────────┘
           │          │          │          │
    ┌──────▼──┐ ┌────▼───┐ ┌───▼────┐ ┌───▼─────┐
    │ 需求设计 │ │代码分析 │ │代码实现 │ │测试执行  │
    │  Agent  │ │ Agent  │ │ Agent  │ │  Agent  │
    └─────────┘ └────────┘ └────────┘ └──────────┘
```

## 设计原则

### 1. 绝对分工
- **主Agent**: 只做调度、汇总、展示。永远不直接操作代码。
- **子Agent**: 只做执行。接收结构化上下文，自行读取文件，返回结构化结果。

### 2. 路径传递
- 主Agent传文件路径，子Agent自行读取
- 子Agent返回文件路径，主Agent不读内容
- 绝对不传代码原文

### 3. Token 预算
- 主→子 ≤500 tokens（结构化 JSON）
- 子→主 ≤300 tokens（结构化 JSON）
- 超出预算 → 拆分任务，不能截断

### 4. 独立上下文
- 每个子Agent启动时传入完整的结构化上下文
- 子Agent不依赖主Agent的对话历史
- 子Agent可自行搜索补充知识

## Agent 数量决策

| 技能规模 | Agent数量 | 示例 |
|----------|----------|------|
| 小型 | 3-5 | 需求设计 + 代码实现 + 测试执行 |
| 中型 | 6-10 | + 脚手架 + 代码分析 + 调试 + 重构 + UI构建 + 测试设计 |
| 大型 | 10+ | + 云服务 + 安全审查 + 性能优化 + 文档生成 |

## Agent 定义标准结构

每个子Agent文件必须包含6个段落：
1. **角色** (Role): 一句话描述专业领域和任务
2. **输入** (Input): 结构化字段表（与协议对齐）
3. **执行** (Execution): 编号步骤，每步可操作
4. **输出** (Output): 结构化 JSON 示例（符合协议）
5. **约束** (Constraints): 编号列表，具体可验证
6. **示例** (Example): 一个真实的输入→输出示例

## 工作流数量决策

| 用户意图类型 | 工作流数量 | 覆盖 |
|-------------|-----------|------|
| 创建/初始化 | 1 | 项目脚手架 |
| 开发功能 | 1-3 | 功能开发、组件开发、服务开发 |
| 测试 | 1-2 | 单元测试、端到端测试 |
| 调试修复 | 1-2 | Bug修复、紧急修复 |
| 优化 | 1 | 性能优化 |
| 审查 | 1-2 | 代码审查、技术选型 |
| 查询 | 1 | 知识查询 |

## 通信模式

### 同步串行
```
主Agent → 启动子Agent A → 等待返回 → 启动子Agent B → 等待返回
```
适用：前后依赖的任务（需求设计 → 代码实现）

### 并行
```
主Agent → 启动子Agent A ─┐
         → 启动子Agent B ─┤ → 等待全部返回 → 合并检查
         → 启动子Agent C ─┘
```
适用：独立任务（UI构建 + 逻辑实现）

### 循环迭代
```
主Agent → 启动子Agent → 检查结果 → 不达标 → 补充指令 → 同一子Agent → ...
```
适用：测试修复、重构改进

## 反模式

| 反模式 | 表现 | 后果 |
|--------|------|------|
| 主Agent越权 | 主Agent直接读/写代码 | 破坏分工，上下文膨胀 |
| 子Agent饥饿 | 子Agent收到不完整上下文 | 输出偏离预期 |
| 字段漂移 | Agent输出与协议不一致 | 主Agent无法解析结果 |
| 无限循环 | 修复→失败→修复无上限 | 资源浪费 |
| 单Agent过载 | 一个Agent负责过多领域 | 浅层输出，质量下降 |
