# 知识查询工作流

纯文档查询，无代码操作。主Agent直接处理，不启动子Agent。

## 触发条件

| 关键词 |
|--------|
| "如何使用XXX"、"XXX怎么用"、"查XXX文档" |
| "XXX的API"、"XXX是什么"、"帮我了解XXX" |
| "查询XXX"、"XXX有哪些限制"、"XXX的最佳实践" |

## 流程

```
用户: "如何在小程序中使用蓝牙"
  │
  ├─ ► 步骤1: 提取搜索关键词 [主Agent]
  │     从用户问题中提取2-3个精准关键词
  │     例: "如何使用蓝牙" → "蓝牙 BLE 设备连接"
  │
  ├─ ► 步骤2: 语义搜索 [主Agent]
  │     执行: python tools/search_docs.py "<关键词>" --top 5
  │     如涉及设计规范: 添加 --category design
  │
  ├─ ► 步骤3: 加载文档 [主Agent]
  │     根据搜索结果，Read 最相关的2-3个 .md 文件
  │     - 得分 > 0.5: 直接加载
  │     - 得分 < 0.3: 补充搜索或查看 FRAMEWORK_INDEX.md
  │
  └─ ► 步骤4: 输出回答 [主Agent]
        结构化输出:
        - 问题直接答案
        - 关键API/配置/限制
        - 代码示例（如果文档包含）
        - 出处: 引用具体的官方文档路径和URL
```

## 搜索技巧

- 用完整的中文短语，避免单个英文缩写
- 如结果不理想，用同义词重新搜索
- 涉及两方面的查询（如"蓝牙+音频"），分别搜索
- category 过滤: framework 用于API，design 用于UI规范

## 输出格式

```markdown
## 回答: [用户问题]

[直接答案，从文档提取，标注出处]

## 关键信息

- API/方法: [xxx] — 出处: framework/xxx.md
- 配置项: [xxx] — 出处: framework/xxx.md
- 限制/注意: [xxx]

## 相关文档

- [标题](路径) — 匹配度: 0.XX
- ...
```

## 约束

- 只查文档不写代码
- 所有回答必须标注官方文档出处
- 搜索结果不理想时主动告知用户并建议查阅官网
- 不要杜撰不存在的API或参数
